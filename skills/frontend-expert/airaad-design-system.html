A1
Design Principles

Four non-negotiable principles govern every design decision in the AirAd portal.

Principle 01
Unified
Consistent experience across all surfaces — same tokens, patterns, interactions.
→ Same component library across all 10 portal pages. Never one-offs.
Principle 02
Universal
Accessible and inclusive by default — WCAG 2.1 AA everywhere.
→ Keyboard nav, screen reader labels, 4.5:1 contrast on all body text.
Principle 03
Iconic
Simple, bold, and purposeful — no decoration for decoration's sake.
→ Whitespace, clear hierarchy, purposeful color. No gratuitous gradients.
Principle 04
Conversational
Design feels human — warm, clear, direct. Not robotic or clinical.
→ Form labels as questions. Errors with guidance, not codes.

A14
Writing Style — Voice & Tone

Sentence case. Direct. Specific. Human. Never technical codes as error messages.

Error messages
✕ "Error 422: Unprocessable Entity"
✓ "GPS accuracy must be 10m or less"
Confirmation messages
✕ "Operation completed successfully"
✓ "Vendor approved"
Empty states
✕ "No records found"
✓ "No vendors match your filters"
Destructive confirmations
✕ "Are you sure? This cannot be undone."
✓ "Remove this vendor? This will archive their data."
Button labels
✕ "Submit" / "OK" / "Yes"
✓ "Save vendor" / "Approve" / "Import CSV"

A11
Accessibility — WCAG 2.1 AA

Non-negotiable. All requirements apply to every page and component.

Contrast Ratios
Body text ≤18px 4.5:1 min ✓
Foggy (#767676) on white 4.48:1 ✓ passes AA
Large text / UI components 3:1 min ✓
Rausch (#FF5A5F) on white 3.04:1 — large/icons only
Key Requirements
  • Critical All interactive elements reachable via Tab in logical order
  • Critical 2px Rausch focus ring on ALL focused elements
  • Critical Focus trapped inside open modals
  • Required aria-live="polite" for all toast notifications
  • Required Minimum touch target 44×44px
  • Note prefers-reduced-motion: disable ALL animations
